Repeterer forfatter-eksempelet fra fredag:

Lager en klasse Forfatter med attributtene fnavn (fornavn), enavn (etternavn) og faar (fødselsår) og metoden visInfo som skriver ut en setning om forfatteren. Forfatterene legges til som objekter og informasjonen skrives ut etter at klassen er definert:
   class Forfatter{ // oppretter klasse
     constructor(fnavn, enavn, faar){ // attributtene til klassen
       this.fnavn = fnavn; // attributtet fornavn får paramenterverdien fornavn
       this.enavn = enavn;
       this.faar = faar;
     }
     visInfo(){// metoden som skriver ut info
       document.write(this.fnavn + " " + this.enavn + " er født i " + this.faar); // this peker på hvilken forfatter
     }
   }
   let blixen = new Forfatter("Karen", "Blixen", 1885); //legger til objekt
   let hamsun = new Forfatter("Knut", "Hamsun", 1859); //legger til objekt
   blixen.visInfo();
   hamsun.visInfo();

Get-metoder

I eksempelet over ble det definert en metode for å skrive ut informasjon om objektene. Forskjellen på en slik, vanlig, metode og en get-metode er at get metoder defineres med get metodeNavn() og har en returverdi. Derfor skal vi ikke inkludere () når metoden kalles.

Vanlig metode:
   class Klassenavn{
     contructor (parameter1, parameter2){
       this.attributt1 = parameter1;
       this.attributt2 = parameter2;
     }
     metodeNavn(){
       //kode metoden skal utføre
     }
   }
   let objektNavn = new Klassenavn(parameter1, parameter2);
   objektNavn.metodeNavn();

   class Forfatter{ // oppretter klasse

       this.fnavn = fnavn; // attributtet fornavn får paramenterverdien fornavn
       this.enavn = enavn;
       this.faar = faar;

Klasser, subklasser og arv

Når man inkluderer subklasser, blir objektorientert programmering enda mer nyttig. En subklasse arver alle parametre og egenskaper fra superklassen, og i tillegg legger vi til egne parmetre og metoder. I klasser kan vi også referere til objekter fra andre klasser på samme måte som når vi ellers har satt verdier til attributtene: vi setter inn objekter som disse verdiene:
   class Klassenavn{
     contructor (parameter1, parameter2){
       this.attributt1 = parameter1;
       this.attributt2 = parameter2;
     }
   }
   class Klassenavn2{
     contructor (parameter1, parameter2){ // her er parameter2 et objekt i klasse 1
       this.attributt1 = parameter1;
       this.attributt2 = parameter2; // her er parameter2 et objekt i klasse 1
     }
   }
   let objektNavn1 = new Klassenavn1(parameter1, parameter2);
   let objektNavn2 = new Klassenavn2(parameter1, objektNavn1);

Eksempel: lager en klasse for bøker som har med forfatter som et attributt
   class Bok{ // definerer klassen bok
     constructor(tittel, forfatter, pris){
       this.tittel=tittel;
       this.forfatter=forfatter;
       this.pris=pris;
     }
   }
   let mittAfrika = new Bok("Mitt Afrika", blixen, 199);
   let fortellinger = new Bok("Syv fantastiske fortellinger", blixen, 150);
   document.write(fortellinger.tittel +" og "+ mittAfrika.tittel + " er skrevet av " + fortellinger.forfatter.enavn + " som ble født i " + fortellinger.forfatter.faar);


Vi kan også relatere klasser sammen ved subklasser og superklasser. Dette bruker vi for å unngå å opprette to nesten like klasser dersom to klasser har mye til felles. Subklassen arver alle attributter fra superklassen, men kan også ha egne metoder som ikke finnes i superklassen. Dersom subklassen har egne versjoner av superklassens metoder, overstyrer subklassens metoder superklassens. Koden for å opprette superklasser og subklasser er:

   class Superklasse{
     constructor (parameter1, parameter 2){
       this.attributt1=parameter1;
       this.attributt2=parameter2;
     }
   metode(){//kode}
   }

   class Subklasse extends Superklasse{
     constructor (parameter1, parameter 2, parameter3){
       super(parameter1, parmeter2); // klassen arver disse to fra superklassen
         this.attributt3=parameter3; // denne er ny for subklassen
     }
     metode(){//kode som likner superklassens metode, men overstyrer}
     egenMetode(){//kode som bare finnes i subklassen}
   }
